# Life Manager - System Documentation

## Overview
Life Manager is a cross-platform Flutter application designed to help users manage their daily tasks and scheduled events efficiently. It features persistent storage, theme customization, and an automated alert system.

## Core Components

### 1. Data Model (lib/models/task.dart)
The `LifeItem` class is the foundation of the app. It uses an `ItemType` enum to distinguish between:
- **Task**: A simple to-do item with a title and description.
- **Event**: A time-sensitive item that includes a specific Date and Time.

### 2. State Management (lib/providers/task_provider.dart)
The app uses the **Provider** pattern for state management. The `TaskProvider` class:
- Maintains the master list of `LifeItem` objects.
- Handles all CRUD (Create, Read, Update, Delete) operations.
- Manages the application's `ThemeMode` (Light, Dark, or System).
- **Persistence**: Automatically saves/loads data using `shared_preferences`.
- **Alert System**: Uses `flutter_local_notifications` to schedule alerts exactly 15 minutes before an event starts.

### 3. User Interface (lib/screens/)
- **LoadingScreen**: Provides a professional entry experience with the app's tagline.
- **TasksScreen**: The main dashboard. It uses `CustomScrollView` and `Slivers` to separate items into "Active" and "Completed" sections.
- **SettingsScreen**: Allows the user to toggle themes and perform a full system reset.

## Code Flow & Logic

1. **Initialization**:
   - `main.dart` initializes `WidgetsFlutterBinding`.
   - `TaskProvider` is created, which immediately loads saved data from storage and initializes the notification engine.

2. **Adding an Item**:
   - User clicks the '+' FAB in `TasksScreen`.
   - A dialog appears where they choose between 'Task' or 'Event'.
   - Upon clicking 'Add', `addItem()` is called in the provider.
   - If it's an event, `_scheduleNotification()` is triggered to set a local alert.
   - Data is saved to local storage, and `notifyListeners()` refreshes the UI.

3. **Updating Status**:
   - When a checkbox is tapped, `toggleItemStatus()` moves the item between the "Active" and "Completed" sections instantly.

4. **Alerts**:
   - The app uses `timezone` data to ensure alerts fire correctly regardless of the user's location.
   - Notifications are scheduled with `exactAllowWhileIdle` to ensure they work even when the phone is in a low-power state.

## System Requirements (Android)
- **Java Desugaring**: Enabled in `build.gradle` to support modern Date/Time APIs on older Android versions.
- **Permissions**: Requires `POST_NOTIFICATIONS` and `USE_EXACT_ALARM` to function correctly on Android 13 and above.
